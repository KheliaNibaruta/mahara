SAML 2.0 field for affiliation IDs

The Identity Provider (IdP) can pass in affiliation information for account holders. This field works in conjunction with

  • SSO field for affiliation emails
  • SSO field for affiliation roles
  • SSO field for affiliation role delimiter

If you set this value, the username is checked and then any affiliated IDs until one is found that already exists in the system. If none are found, the username of the main identity is used as remote username.
