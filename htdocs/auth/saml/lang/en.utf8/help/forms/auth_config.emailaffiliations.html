SAML 2.0 field for affiliation emails

The Identity Provider (IdP) can pass in affiliation information for account holders. This field works in conjunction with

  • SSO field for affiliation IDs
  • SSO field for affiliation roles
  • SSO field for affiliation role delimiter

If you set this value, the system will check which affiliation role is the highest and set the associated email to be the primary email address when the account holder logs in for the first time.

If you leave this blank then the value in 'SSO field for email' will be used instead.
